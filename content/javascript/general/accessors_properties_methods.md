## Свойства и методы доступа в JavaScript

В JavaScript объекты играют ключевую роль, позволяя хранить данные в виде пар "ключ-значение". Доступ к этим данным, их изменение и управление ими осуществляется с помощью свойств и методов доступа.

### Свойства данных

Свойства данных – это, по сути, переменные, которые хранятся внутри объекта. Они содержат информацию, связанную с этим объектом. Например, у объекта "книга" могут быть свойства данных "название", "автор" и "год издания".

Давайте создадим объект "книга" и добавим ему некоторые свойства:

```javascript
const книга = {
  название: "Мастер и Маргарита",
  автор: "Михаил Булгаков",
  годИздания: 1967
};
```

Теперь мы можем получить доступ к значениям этих свойств, используя точечную нотацию или скобочную нотацию:

```javascript
console.log(книга.название); // Вывод: "Мастер и Маргарита"
console.log(книга["автор"]); // Вывод: "Михаил Булгаков"
```

Мы также можем изменить значение свойства:

```javascript
книга.годИздания = 1966; // Исправляем год издания
console.log(книга.годИздания); // Вывод: 1966
```

### Методы доступа: Getters и Setters

В JavaScript свойства не ограничиваются только хранением данных. С помощью специальных методов, называемых getters и setters, можно контролировать доступ к свойствам, а также выполнять дополнительные действия при чтении или изменении значений.

**Getters**

Getters – это функции, которые вызываются при чтении значения свойства. Они позволяют выполнить вычисления или преобразовать данные перед их возвратом.

Рассмотрим пример:

```javascript
const прямоугольник = {
  ширина: 5,
  высота: 10,
  get площадь() {
    return this.ширина * this.высота;
  }
};

console.log(прямоугольник.площадь); // Вывод: 50
```

В этом примере `площадь` – это getter. При обращении к `прямоугольник.площадь` вызывается функция getter, которая вычисляет и возвращает площадь прямоугольника.

**Setters**

Setters – это функции, которые вызываются при изменении значения свойства. Они позволяют выполнять валидацию, преобразование данных или другие действия перед тем, как новое значение будет присвоено свойству.

```javascript
const человек = {
  _имя: "", // Приватное свойство с префиксом "_"
  set имя(новоеИмя) {
    if (typeof новоеИмя === "string" && новоеИмя.length > 0) {
      this._имя = новоеИмя;
    } else {
      console.log("Ошибка: Некорректное имя.");
    }
  },
  get имя() {
    return this._имя;
  }
};

человек.имя = "Иван"; // Вызов setter
console.log(человек.имя); // Вывод: "Иван"

человек.имя = 123; // Попытка установить некорректное имя
console.log(человек.имя); // Вывод: "Иван" (значение не изменилось)
```

В данном примере setter `имя` проверяет, является ли новое значение строкой и не пустой ли она. Если проверка проходит успешно, новое имя устанавливается в приватное свойство `_имя`. В противном случае выводится сообщение об ошибке.

### Преимущества использования getters и setters

* **Инкапсуляция**: Getters и setters позволяют скрыть внутреннюю реализацию объекта и предоставить контролируемый интерфейс для взаимодействия с его данными.
* **Валидация данных**: Setters могут использоваться для проверки корректности устанавливаемых значений.
* **Вычисляемые свойства**: Getters позволяют создавать динамические свойства, значения которых вычисляются на лету.

### Заключение

Свойства и методы доступа играют важную роль в работе с объектами в JavaScript. Понимание принципов их работы позволяет создавать более гибкие, безопасные и удобные в использовании приложения.
